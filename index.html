「Webコンテンツ作成」の成果

本テーマについて

Webコンテンツ作成の目的としては、情報工学の知識を応用して、実践的な能力を修得することである。目標については2つある。1つ目は手順書を細かく読んで実験を行った後に、レポートに報告するという実験の一連の流れを実践できるようになることである。2つ目はHTMLやCSS、JavaScript等を用いたWebページを作成し、課題に対する解決能力の育成に寄与することである。

プロジェクト課題について

私が今回のWebコンテンツ作成の授業で取り組んだプロジェクト課題の内容は、世界の一部の国の都市の天気を調べることが出来る内容のプロジェクト課題である。このプロジェクト課題では、テキストボックスに指定されている数字のみの都市IDを入力して、その後に検索ボタンを押すことによって、自分が入力した都市IDの都市名とその都市の天気の情報だけではなく、その都市の経度と緯度、最低気温と最高気温、さらに湿度と風速、風向までの情報が画面に表示される。加えて、その都市の天気の種類によって、それぞれの天気に応じた画像が都市名の下に大きく表示される。天気の画像の特徴として、晴れの天気の場合はオレンジ色の太陽の画像が表示され、厚い雲や曇りがち、さらに霧の天気の場合は白色の雲の画像が表示される。他にも小雨の天気の場合は青色の傘と長い雨粒の画像が表示される。

プロジェクト課題で作成したページ：project.html

数当てゲーム

この数当てゲームでは、事前にコンピュータ側が乱数で1から10までの値を1つ決めており、それに対してユーザ側はコンピュータ側が決めた1から10までの値が何かを予想しなければならない。次にユーザ側はコンピュータ側が決めた値を予想することが出来たら、その予想した値の答えをテキストボックスに入力する。ここでコンピュータ側が決めた値をユーザ側が当てることに成功したら、ユーザ側の勝利である。しかし、ユーザ側がコンピュータ側が決めた値を外してしまった場合は、また1から10までの違う値を予想し、テキストボックスに入力しなければならない。これを3回まで繰り返し、3回までにコンピュータ側が決めた値をユーザ側が予想して当てることが出来なかった場合、コンピュータ側の勝利である。

数当てゲームのページ：kazuate.html

各授業回の学習事項

第1回 WWW と HTML

私が今回の講義で学習した事柄について、最初はHTMLについての概要を理解した後に、文字の入力の仕方や画像をWebページに追加する方法、ハイパーリンクの設置の仕方を学びました。さらに特殊文字やリストを作る方法についても、不手際なく理解することが出来ました。

演習の成果：演習1-4：HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

私が今回の講義で学習した事柄について、最初はCSSの概要について理解した後に、文字に色を追加する方法や箇条書きの点をなくす方法、リンクの色の設定の仕方やプロパティ値についても学びました。そして私は前回学んだHTMLと今回学んだCSSを使い、綺麗で彩りのあるWebページを作ることが出来ました。

演習の成果：演習2-3：ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

私が今回の講義で学習した事柄について、最初はJavaScriptの概要について学び、次にJavaとJavaScriptのプログラムの細かい違いについて学びました。その次にJavaScriptの構文について詳しく学び、最後にウェブ開発ツールについてのデバッグの仕方や、エラー発生時のプログラムの実行の一時停止方法についても学び、JavaScriptの基礎的な学習を深めることが出来ました。

演習の成果：演習3-6：オブジェクトの参照

第4回 DOM と JavaScript

私が今回の講義で学習したことは三つある。一つ目はinput要素である。HTMLとJavaScriptを利用し、入力フォームのページの作り方について学んだ。二つ目はDOMについてである。Webコンテンツの内容を注意深く読み取り、コンテンツに変更を加える処理の書き方について学びました。三つ目はイベント処理についてである。イベントハンドラの登録と外部プログラムを記述することについての学びを深めることが出来ました。

演習の成果：演習4-10：足し算ページ

第5回 イベント処理

私が今回の講義で学習したことは三つある。一つ目はDOMによる要素の新規作成についてである。新規要素のオブジェクトを作成し、HTMLにその新規要素を配置することについて学んだ。二つ目はDOMによる要素の編集についてである。要素の属性を追加したり、変更したりすることについて学びました。三つ目はDOMによる要素の削除についてである。コンソールを使って箇条書きの文字を消したり、箇条書き自体を削除したりすることについての学びを深めることが出来ました。

演習の成果：演習5-7：DOMプログラミング(2)

第6回 通信処理

私は今回の講義でAjax通信による通信処理について学びました。Ajax通信はリクエストを送った後に、ブラウザの別の処理を実行することが可能であり、レスポンスの受け取りはイベント処理としての記述が出来ることを学びました。さらにHTMLとJavaScriptプログラムを利用して、リクエストの送信やレスポンスの受け取り、さらにエラー処理についてもAjax通信の実例を通して深く学べました。

工夫点，苦心点，強調点，感想

私がこれまでの授業で感じたことについての、工夫点と苦心点、強調点と授業全体を通した感想の四つを述べる。最初に、工夫点について述べる。工夫点としてはHTMLとCSS、加えてJavaScriptのコードを一つ一つより細かく見て、誤字脱字がないかを細部までしっかりと確認したことである。詳細として、私は毎回の演習課題のWebページが完成しても、すぐにリポジトリへの登録をせずに、10分以上誤字脱字がないかを時間かけて確認しました。次に、苦心点について述べる。苦心点としてはコンソールによるJavascriptのコードを打ったことである。特にコンソールでJavascriptのコードを打つ時に、一文字でも誤字脱字をしたら、最初からコードを打ち直して、やり直しになる機会がこの授業では非常に多かったからである。さらに、強調点について述べる。強調点としては多様な色を使い、項目を文字の色ごとに分けた点である。特にWebページの中で、私自身が強調して伝えたい項目は、なるべく濃い色を選ぶことを心がけており、それ以外の配色の面では、薄くてよりおしゃれな色を使うように心がけることを意識しました。Webページ全体を色鮮やかにすることによって、見やすくて綺麗なWebページに知ることが出来たと感じている。最後に、授業全体を通した感想を述べる。私は今回のWebコンテンツ作成の授業全体を通して、Webページを作れる能力を身につけるようになっただけではなく、普段私自身が日常生活で使用しているWebがどのような仕組みなのだろうかと興味を持てました。私はWebコンテンツ作成の授業を受ける前までは、Webページのことについて全く興味が持てずにいましたが、Webコンテンツ作成の授業を受けていくうちに、普段の日常生活で使用しているWebを利用する時に、自分が普段のWebコンテンツ作成の授業で学んでいることと深く結びついているのだろうと感じることが出来るようになり、よりWebページのことについて興味が持てるようになりました。さらに普段のサークル活動でも、今回のWebコンテンツ作成の授業で学んだことを活用して、自力で新しくWebページを作りたいと感じました。自分の力でWebサイトを作れるようになった影響もあり、私自身が立派なエンジニアになれる未来像が、少しずつ見れるようになり、今回のWebコンテンツ作成の授業を受けて、普段の情報工学の専門科目の勉強を、より一層頑張りたいと思えるようになりました。
